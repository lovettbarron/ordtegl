<task type="auto">
  <name>Task 2: Create SASS design system with warm neutrals palette</name>
  <files>_sass/_variables.scss, _sass/_base.scss, assets/css/main.scss</files>
  <action>
Create `_sass/_variables.scss` with Ordtegl warm neutrals design tokens:

Color palette:
- $color-cream-light: #FAF7F2 (background)
- $color-cream: #F5EFE7 (surface/cards)
- $color-gray-soft: #D8D4CF (borders)
- $color-gray-medium: #8E8B87 (secondary text)
- $color-gray-dark: #4A4745 (primary text)
- $color-terracotta: #D4826C (accent/links)
- $color-terracotta-dark: #B8654F (hover states)

Semantic tokens mapping palette colors to purposes:
- $color-background, $color-surface, $color-text-primary, $color-text-secondary
- $color-accent, $color-accent-hover, $color-border

Typography tokens:
- $font-family-base: system font stack (-apple-system, BlinkMacSystemFont, etc.)
- $font-size-base: 16px
- $line-height-base: 1.6

Spacing scale:
- $spacing-xs through $spacing-xl (0.5rem to 6rem)

Layout tokens:
- $max-width-content: 800px (for legal pages readability)
- $border-radius-sm and $border-radius-md

Create `_sass/_base.scss` with base element styles:
- body: background, color, font-family, line-height
- h1-h6: color, line-height, margin
- a: color, hover state
- p: margin-bottom for readability
- ul, ol: proper spacing for legal documents

Create `assets/css/main.scss` (must have empty front matter for Jekyll to process):
```scss
---
---

@import "variables";
@import "base";
```

The main.scss must have the triple-dash front matter or Jekyll won't process it.
  </action>
  <verify>
Run `bundle exec jekyll build` - should compile SASS to CSS without errors.
Check `_site/assets/css/main.css` exists and contains the color values.
  </verify>
  <done>SASS files exist, compile successfully, CSS output contains warm neutrals colors</done>
</task>
